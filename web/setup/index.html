Local Backend Setup

The AI PM Assistant runs Claude generation in your browser via Vercel. To use Jira and Confluence features, you also need the local Flask bridge running on your machine.

What runs where: AI generation (Claude/Haiku) happens via Vercel — no local server needed for that. The local backend only handles Atlassian OAuth and Jira/Confluence REST calls.
Installation steps
1 Prerequisites

Make sure you have the following installed on your machine:

  • Python 3.9+ — python.org/downloads
  • pip — bundled with Python 3.9+

Verify your Python version:

python --version
2 Get the files

Clone the repository or download and extract the ZIP to a folder on your machine.

git clone <repo-url> cd AIPMAssistant

Or download the ZIP, extract it, and open the folder in your terminal.

3 Install Python dependencies

From the project root folder, run:

pip install -r requirements.txt

This installs Flask, flask-cors, requests, and other dependencies.

4 Create your .env file

Create a file named .env in the project root (same folder as app.py):

ATLASSIAN_CLIENT_ID=your_client_id_here ATLASSIAN_CLIENT_SECRET=your_client_secret_here SECRET_KEY=any_random_string_here

Required variables:

Variable Description
ATLASSIAN_CLIENT_ID OAuth 2.0 Client ID from developer.atlassian.com
ATLASSIAN_CLIENT_SECRET OAuth 2.0 Client Secret from developer.atlassian.com
SECRET_KEY Any random string — used for Flask session encryption
Never commit your .env file to git. It's already in .gitignore.
5 Get Atlassian OAuth 2.0 credentials

Go to developer.atlassian.com/console/myapps/ and create a new app:

  1. Click Create → OAuth 2.0 integration
  2. Give it a name (e.g. "AI PM Assistant")
  3. Under Authorization, add callback URL: http://localhost:5001/oauth/callback
  4. Under Permissions, enable: Jira (read/write), Confluence (read/write), User identity
  5. Copy the Client ID and Secret into your .env
The required OAuth scopes are: read:jira-work write:jira-work read:jira-user read:confluence-content.all write:confluence-content read:page:confluence read:space:confluence write:page:confluence offline_access read:me
6 Start the backend

Option A — double-click start.bat (Windows)

Option B — run from terminal:

python app.py

You should see:

============================================================ AI PM Assistant — http://localhost:5001 ============================================================ Jira (OAuth): not connected — open UI to connect OAuth callback: http://localhost:5001/oauth/callback ============================================================

The backend runs at http://localhost:5001 by default.

7 Connect Jira in the app

Once the backend is running, open the app (back to app) and:

  1. Click ⚙ Settings in the header — verify the Backend URL is http://localhost:5001 and shows "Connected"
  2. Add your Anthropic API key in Settings under "Anthropic API Key"
  3. Click 🔗 Connect to Jira — this opens the Atlassian OAuth flow
  4. Approve the permissions — you'll be redirected back to the app

The header will show your Atlassian user name when connected.

Troubleshooting
Backend shows "Offline" in the app
Make sure python app.py is running and there are no errors in the terminal. Check that the Backend URL in Settings is set to http://localhost:5001. Click "Test & Save" to retry.
Port 5001 is already in use
Another process is using port 5001. Either stop it or set a different port by adding PORT=5002 to your .env file. Update the Backend URL in Settings to match.
OAuth callback error after connecting Jira
Make sure the callback URL in your Atlassian app matches exactly: http://localhost:5001/oauth/callback. Also verify ATLASSIAN_CLIENT_ID and ATLASSIAN_CLIENT_SECRET are correct in your .env.
403 Forbidden when pushing to Jira/Confluence
Your OAuth token is missing write scopes. Disconnect from Jira in the header, then reconnect — this will request fresh permissions including write access.
AI generation not working
Add your Anthropic API key in Settings (⚙ in the header). Get a key at console.anthropic.com. The key is stored only in your browser and never sent to anyone except Anthropic.
pip install fails
Try pip3 install -r requirements.txt. On some systems Python 3 is python3 and pip is pip3.
